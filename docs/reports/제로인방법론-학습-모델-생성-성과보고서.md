## 1. 사업 개요

본 사업은 도메인 특화 한국어 대규모 언어모델(LLM)을 자체 구축하기 위해 정부의 GPU 임차
지원을 활용하여 수행된 연구개발 과제이다. 공개 베이스 모델 `openai/gpt-oss-20b`(Apache 2.0)
위에, 한국 자산운용업계의 펀드 평가 도메인 지식(제로인 펀드평가 방법론)을 풀
파인튜닝(full fine-tuning) 방식으로 주입하고, 그 결과 모델과 학습 데이터셋을 비상업
오픈 라이선스(CC BY-NC 4.0)로 Hugging Face Hub에 공개하였다.

## 2. 추진 목표

- 한국어·도메인 특화 LLM의 풀 파인튜닝 파이프라인을 H200 다중 GPU 환경에서 안정적으로 구축
- 자체 도메인 코퍼스(제로인 펀드 평가 방법론)를 Harmony 채팅 포맷의 학습 데이터셋으로 정비
- 일별 재학습 → 산출물 자동 서빙(vLLM) → OpenWebUI를 통한 외부 사용자 검증의 end-to-end 자동화
- 모델·데이터셋의 공개(Open) 배포를 통한 한국어 도메인 LLM 생태계 기여

## 3. 추진 내용 및 시스템 구성

### 3.1 학습 환경
- **하드웨어**: NVIDIA H200 × 3 (총 ~440 GB HBM3e VRAM)
- **소프트웨어 스택**: Hugging Face transformers + TRL + DeepSpeed ZeRO-3, BF16
- **베이스 모델**: `openai/gpt-oss-20b` (MoE, 32 experts / 4 active, ~21B 파라미터)

### 3.2 데이터셋 ([`KG-ZEROIN/zeroin-v8-r4-dataset`](https://huggingface.co/datasets/KG-ZEROIN/zeroin-v8-r4-dataset))
- 원본: 제로인 펀드 평가 방법론 한국어 도메인 CSV ([`v9-r5.csv`](artifacts/data/v9-r5.csv))
- 가공: 다중 질문 변형(Q/A 그룹별 paraphrasing) 확장 → Harmony 메시지 JSONL ([`seed_v8_round4_section_all_full_ft.jsonl`](artifacts/data/seed_v8_round4_section_all_full_ft.jsonl))
- 최종 학습 코퍼스: **2,638 records** (236 Q/A groups, 15,240 expanded variants)
- 라이선스: **CC BY-NC 4.0**

### 3.3 파인튜닝 파이프라인
- 풀 파인튜닝 (모든 파라미터 학습, MoE 구조 포함)
- 학습 설정: max_steps 1000, lr 1e-5 (cosine + warmup), per-device bs 1, grad accum 8 (effective bs 24)
- 자동화 daemon ([`nightly_v8_round4_twice.sh`](artifacts/daemons/nightly_v8_round4_twice.sh)): 매일 18:00 KST에 동일 데이터셋으로 RUN 1 → RUN 2 → RUN 3 (총 ~17h 36m) 순차 실행, 결과 덮어쓰기 방식
- 학습 산출물 검증 자동화 ([`run_full_ft_gpt_oss_20b.py`](artifacts/scripts/run_full_ft_gpt_oss_20b.py) `--orchestrate`)

### 3.4 서빙 환경
- **vLLM serve** (BF16, max-model-len 131072 / 학습 시 4096) — 매일 12:00~17:50 KST 자동 기동 ([`daily_vllm_serve.sh`](artifacts/daemons/daily_vllm_serve.sh), [`run_vllm_model_server.sh`](artifacts/scripts/run_vllm_model_server.sh))
- **OpenWebUI** (포트 8080) — 외부 사용자 검증 인터페이스
- **ngrok tunnel** (`thundering-unaisled-serita.ngrok-free.dev`) — 외부망 접속 노출
- 학습 시간대(18:00~익일 ~11:36)에는 vLLM이 자동 종료되어 GPU 자원 충돌 방지

### 3.5 운영 자동화
- 학습 daemon, vLLM serving daemon이 각각 `while true` 루프로 시간 윈도우 기반 동기화
- 일별 학습 사이클 성공률 100% (분쟁 구간 없음)
- 모델 가중치 매일 갱신, 동일 경로의 `final-export/`에 덮어쓰기

## 4. 주요 성과

### 4.1 공개 산출물 (Hugging Face Hub, KG-ZEROIN 조직)

| 종류 | Repository | 라이선스 |
|------|------------|----------|
| Fine-tuned model (safetensors) | [`KG-ZEROIN/gpt-oss-20b-zeroin-v8-r4`](https://huggingface.co/KG-ZEROIN/gpt-oss-20b-zeroin-v8-r4) | CC BY-NC 4.0 |
| Training dataset | [`KG-ZEROIN/zeroin-v8-r4-dataset`](https://huggingface.co/datasets/KG-ZEROIN/zeroin-v8-r4-dataset) | CC BY-NC 4.0 |

### 4.2 정량 지표
- 학습 데이터: **2,638 records** (~7.5 MB, JSONL) + 원본 CSV (~841 KB)
- 모델 파일: **41.83 GB** (BF16 safetensors)
- 학습 횟수: **약 60+ 사이클** (4/17 ~ 6/25, 일별 2회 또는 3회)
- 학습 종료 시점 손실: **~0.003** (RUN 3 종료 시점 기준)

### 4.3 정성 성과
- 한국어 펀드평가 도메인 특화 Q&A 능력 확보 (예: 충분성/비교성/지속성 원칙, 위험조정 성과, 샤프지수, IR, 스타등급, %순위 등)
- Harmony 채팅 포맷 자동 처리 (analysis → final 채널 구조 보존)
- OpenWebUI 외부 검증 환경 구축으로 비전문가 사용자도 직접 모델 품질 검증 가능

## 5. 주요 기술적 도전과 해결

| 이슈 | 원인 | 해결 |
|------|------|------|
| `--orchestrate` 모드 첫 실행 실패 | 시스템 `python`이 `/usr/bin/torchrun`을 못 찾음 | 학습 스크립트가 venv python 절대경로를 강제하도록 수정 |
| vLLM 스트리밍 응답 빈 content | vLLM 0.13이 `model_type=gpt_oss`에 harmony 파서를 강제 적용, 파인튜닝 모델은 채널 마커 없이 출력 | `VLLM_DISABLE_HARMONY=1` env var로 파서 비활성화 (vLLM 패치 + daemon 스크립트 반영) |
| OpenWebUI MCP 도구 호출 무한 재시도 | gpt-oss-128k 모델이 `tool.query_etf_data`처럼 prefix를 붙여 호출, OpenWebUI 디스패처는 정확 일치만 수행 | `middleware.py`에 prefix tolerant lookup 패치 추가 |
| 컨텍스트 제한 4,096 토큰 | 기본 max-model-len이 작아 긴 대화 차단 | 모델 네이티브 한도 **131,072 토큰**으로 확장 |
| 자동 daemon이 메모리 캐싱된 옛 스크립트로 동작 | bash daemon이 스크립트를 메모리에 로드 후 파일 수정 미반영 | daemon 재기동 절차 표준화, 메모리에 기록 |

## 6. 기대 효과 및 향후 활용

- 한국어 펀드평가 도메인 LLM의 베이스라인 공개 → 후속 연구 및 산업체 활용 가능
- 풀 파인튜닝 → 도메인 데이터 → 서빙까지의 단일 자동화 파이프라인 사례 확보
- 동일 인프라를 활용한 후속 라운드(round5 등) 및 다른 도메인(헤지펀드, ETF 등)으로 확장 용이
