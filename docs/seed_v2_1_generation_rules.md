# Seed v2.1 Generation Rules

## 목표

- `seed_v2.1`은 `seed_v2`를 단순 요약한 버전이 아니라, 원문 구조와 규칙을 더 충실히 반영한 학습 데이터 버전입니다.
- 질문은 자연스럽게 짧게 만들고, 답변은 원문 핵심 내용과 하위 구조를 충분히 담는 방식으로 작성합니다.

## 질문 규칙

- 질문은 실제 사용자가 물을 법한 자연스러운 한국어 문장으로 작성합니다.
- 불필요한 백틱, 인용부호, 문서명 반복, `주제` 같은 메타 표현은 쓰지 않습니다.
- 상위 제목에 대한 질문은 상위 범위를 묻되, 하위 구조까지 답변할 수 있는 형태로 씁니다.

예시:

- `유형분류 기준이 무엇인가요?`
- `국내투자 펀드와 국외투자 펀드의 유형분류 기준은 각각 무엇이며, 어떤 요소로 구분되나요?`

## 답변 규칙

- 답변은 지나치게 압축하지 않습니다.
- 원문 문장을 그대로 길게 복사하기보다, 원문 구조와 핵심 표현을 유지한 설명형 답변을 만듭니다.
- 메타 문구는 금지합니다.

금지 예시:

- `주제:`
- `핵심 주제:`
- `하위 구성:`
- `대상 표:`
- `추가 요소:`
- `포맷:`
- `제약:`
- `해당 세부 기준과 설명이 이어집니다`

## Coverage 규칙

- 상위 질문이면 하위 제목 전체를 답변에 포함합니다.
- 제목 마크다운 기호는 제거해도 되지만, 제목 텍스트는 유지할 수 있습니다.
- 하위 제목 이름만 나열하지 말고, 각 제목 아래 핵심 기준/수치/예외를 함께 설명합니다.

예시 형식:

```text
1.1 유형분류 개요
...

국내/국외 유형분류 기준
...

유형생성의 기본원칙
...
```

## Table Explainer 규칙

- 표가 짧거나 핵심 구분이 중요하면 대표 행을 충분히 나열해도 됩니다.
- 최소한 아래를 포함합니다.
  - 열 이름
  - 기준값 또는 구분 기준
  - 대표 항목
  - 예외 또는 적용 순서
- 일반론으로 바꾸지 말고 실제 값 중심으로 설명합니다.

## Rule / Concept / Definition 규칙

- 정의/기준/예외를 충분히 설명합니다.
- 원문에 있는 수치, 날짜, BM 이름, 기간 조건을 우선 반영합니다.
- 일반 금융 상식으로 둔화시키지 않습니다.

## 품질 체크리스트

- 메타 라벨이 없는가
- placeholder 문장이 없는가
- 상위 질문이 하위 제목 전체를 실제 내용과 함께 포함하는가
- 표 설명에서 열 이름과 기준값이 살아 있는가
- 원문 수치, 날짜, BM 이름, 운용 조건이 누락되지 않았는가
