# V8 Round1 Retry1 산출물 비교 메모

## 비교 대상

- 기존 기준: `tests/log/v8_round1_full_ft_checkpoint1000_quality_report.md`
  - 검증 대상: `checkpoint-1000`
  - generation 설정: `max_new_tokens=256`
- 신규 기준: `tests/log/v8_round1_full_ft_retry1_final_export_report_512.md`
  - 검증 대상: `retry1/final-export`
  - generation 설정: `max_new_tokens=512`

## 핵심 비교

### 1. 운영 성공성

- 기존 기준은 `checkpoint-1000` 자체는 로드 가능했지만, 학습 런 전체는 `final export` 종료 구간의 `NCCL timeout` 때문에 성공 런으로 정리되지 못했다.
- 신규 기준은 `retry1`에서 `checkpoint-1000`과 `final-export` 산출물이 모두 생성되었고, `final-export`를 직접 다시 로드한 validator도 `success`로 끝났다.
- 따라서 이번 `retry1`의 가장 큰 개선은 모델 내용 자체보다도, `final-export`를 실제로 확보하고 그것을 다시 로드해 검증할 수 있는 운영 경로가 살아났다는 점이다.

### 2. 출력 형식 안정성

- 두 결과 모두 Harmony `final` 채널 추출 자체는 정상적으로 동작했다.
- 기존 기준에서도 내부 `analysis` 누수는 보이지 않았고, 신규 기준에서도 같은 안정성이 유지됐다.
- 즉 포맷 측면에서는 기존 Harmony 재설계 효과가 유지됐고, 이번 `retry1`은 이를 깨뜨리지 않았다.

### 3. 답변 완결성

- 기존 기준의 샘플 답변은 여러 프롬프트에서 공통적으로 `정리하면, 유형은 충분한 규모가 있고`에서 끊겼다.
- 신규 기준의 샘플 답변은 같은 설명 구조를 유지하지만 마지막 문장이 `유형은 충분한 규모가 있고(충분성), 서로 비교 가능하며(비교성), 장기적으로 유지될 수 있어야(지속성) 의미 있는 분류로 인정됩니다.`까지 닫힌다.
- 따라서 이번 비교에서 가장 직접적으로 보이는 품질 개선은 "새로운 내용 추가"라기보다 "문장 완결성과 마무리 회수"다.

### 4. 내용 수준

- 기존 기준과 신규 기준 모두 샘플 5개에 대해 거의 동일한 설명 틀을 사용한다.
- 질문 변형이 달라도 답변 구조가 매우 비슷하므로, 현재 모델은 해당 문항군에 대해 안정적으로 한 가지 정리 패턴을 재생산하는 상태에 가깝다.
- 즉 이번 산출물은 "형식 누수 없는 완결 답변"으로는 개선됐지만, "표현 다양성"이나 "질문별 세밀한 적응"이 크게 좋아졌다고 보기는 어렵다.

## 해석

- 좋아진 점:
  - `final-export` 실산출물 확보
  - `final-export` 재로딩 검증 성공
  - `256` 기준에서 보이던 답변 중간 잘림이 `512` 기준 샘플에서는 사라짐
- 그대로인 점:
  - 답변 내용의 기본 골격은 기존과 거의 동일함
  - 질문별 표현 변화 폭은 여전히 작음
- 아직 남은 점:
  - 이번 비교는 `checkpoint-1000` 대 `final-export` 비교이면서, 동시에 `256` 대 `512` generation 상한 차이도 함께 들어가 있으므로, 완결성 개선의 일부는 validator 토큰 상한 증가 영향일 수 있다
  - 러너의 `train_result.json`은 여전히 dry-run 값을 유지하고 있어, 운영 보고서 기록 경로는 따로 손볼 여지가 남아 있다

## 결론

- 이번 `retry1` 산출물은 기존 실패 런 대비 "운영적으로 더 완전한 결과"다.
- 품질 면에서는 내부 채널 누수 없이 끝까지 닫히는 답변을 보여 준다는 점이 가장 분명한 개선이다.
- 반면 답변 내용 자체는 기존과 매우 유사하므로, 이번 결과를 "질적 도약"보다는 "완료 가능한 full fine-tuning + 재로딩 가능한 export + 완결성 개선"으로 해석하는 것이 가장 정확하다.
